GitHub Download Now — Privacy Policy

Effective date: July 14, 2026

GitHub Download Now does not sell user data and does not send data to the developer, analytics providers or advertising services.

Public repositories only

The extension is designed for public GitHub repositories. It does not request GitHub authentication tokens, does not read cookie values and does not intentionally process private repository content. The content script activates only after GitHub provides a positive public-repository marker. It remains inactive on private repositories and fails closed when visibility cannot be established.

Public GitHub page data

The extension reads the public repository identifier, release tags and release-asset links available on the current GitHub page. It may anonymously request public pages at github.com for another selected release version. These requests use credentials: omit, so the browser's GitHub session cookies are not attached.

Release, source archive and navigation URLs are validated against the expected GitHub origin and repository before they are shown, stored or opened.

GitHub REST API

The extension may query the official public API at api.github.com when page data is unavailable, when the user asks to find build documentation, or when checking repositories the user explicitly chose to watch.

For build documentation, it may read public directory listings and likely README text for the selected tag. It analyses Markdown headings such as “Building” and links to the original section. It does not execute build commands or send repository content to any service other than GitHub.

Update checks are rotated in small batches. Rate-limit metadata is stored locally so checks can pause until GitHub's published reset time.

Local installation guidance

Installation and launch guidance is generated locally from the selected filename, extension, detected platform and interface language. It is not sent to an AI service or developer server. Commands are never executed automatically.

Local storage

Optional download records, watched repositories, pending updates, ETags and scheduling metadata are stored in storage.local. Preferences may be synchronized by the browser through storage.sync. The extension does not read the browser's general download history and writes no local history from private/incognito tabs.

Notifications

System notifications are disabled by default and require the optional notifications permission. The toolbar badge does not require notification permission.

Data not accessed

  • cookie values or GitHub authentication tokens;
  • private repository content by design;
  • account profile data;
  • the browser's browsing-history database or general download history;
  • personal communications or unrelated sensitive information.

The extension observes only the currently open GitHub page required for its user-facing controls. It does not query the browser history API.

Third parties and Limited Use

The only network service queried is GitHub, through public pages at github.com and the official API at api.github.com. There is no extension-operated backend, telemetry endpoint, AI service or advertising service.

Information received through browser APIs is used only for release downloads, installation guidance and opt-in update tracking. It is not used for advertising, profiling, creditworthiness, sale to third parties or unrelated purposes, and is not made available for human review by the developer.

User controls

Users can disable the extension, select manual-only checks, remove watches, clear local history, hide installation guidance, disable notifications or uninstall the extension.
